# SS-05: Sentiment & Macro Collector

**Master Spec Sections:** §3.5, §8
**Dependencies:** SS-01
**Estimated LOC:** ~200

## Scope

Collects sentiment data (Fear & Greed Index from Alternative.me) and macro economic calendar data (from static JSON file + FRED API for actuals). Loads macro events into the `macro_events` table, provides functions to query upcoming events and compute hours-until-event. Fetches CPI actuals from FRED API post-release and computes surprise impact classification. This sub-spec does NOT compute Event Risk scores (SS-13), Signal 4 Mean Reversion scores (SS-12), or generate alerts — it only fetches and stores sentiment/macro data.

## Key Requirements

1. `SentimentCollector` class with async methods for API calls, sync methods for local file operations. Reuses `CollectorError` from `custom/collectors/spot.py`.
2. `fetch_fear_greed()` fetches Fear & Greed Index via `GET https://api.alternative.me/fng/`. Returns dict with `value` (int 0–100), `value_classification` (str), and stores in `daily_snapshot` table (updates `fear_greed` column). Returns `None` on failure.
3. `load_macro_calendar()` reads `config/macro_events_2026.json` and bulk-inserts events into `macro_events` table. Skips events that already exist (by date + event name). Returns count of newly inserted events.
4. `get_upcoming_events(hours_ahead: int = 48)` queries `macro_events` table for events within the next N hours. Returns list of dicts with `event`, `tier`, `date`, `time_utc`, `hours_until`. Sorted by `hours_until` ascending.
5. `get_next_event()` returns the single closest upcoming macro event, or `None` if no events within 48 hours. Returns dict with `event`, `tier`, `hours_until`, `date`, `time_utc`.
6. `fetch_cpi_actual()` fetches latest CPI value from FRED API via `GET https://api.stlouisfed.org/fred/series/observations?series_id=CPIAUCSL&sort_order=desc&limit=1&api_key=...&file_type=json`. Requires FRED API key from environment variable `FRED_API_KEY`. Returns `float` or `None` on failure.
7. `classify_surprise(event_type: str, actual: float, forecast: float)` computes surprise impact:
   - For inflation data (CPI, PCE, PPI): `surprise = actual - forecast`
     - `> +0.2` → `VERY_BEARISH`
     - `+0.1 to +0.2` → `BEARISH`
     - `-0.1 to +0.1` → `NEUTRAL`
     - `-0.2 to -0.1` → `BULLISH`
     - `< -0.2` → `VERY_BULLISH`
   - For jobs data (NFP): `surprise = actual - forecast` (thousands)
     - `> +100` → `BEARISH`
     - `+50 to +100` → `NEUTRAL`
     - `-50 to +50` → `BULLISH`
     - `< -50` → `VERY_BEARISH`
   - All thresholds read from `config/settings.yaml` `macro_events.*`
8. `update_event_actual(date: str, event: str, actual: float, forecast: float)` updates the `macro_events` row with `actual`, `forecast`, computed `surprise`, and `impact` classification. Returns the impact string.
9. Fear & Greed API base URL: `https://api.alternative.me/fng/`. FRED API base URL: `https://api.stlouisfed.org`. No rate limit concerns (1 call/day each).
10. All thresholds read from `config/settings.yaml`, not hardcoded.

## Acceptance Criteria

- [ ] `fetch_fear_greed()` returns value (int 0–100) and classification string
- [ ] `fetch_fear_greed()` returns None when API fails (graceful degradation)
- [ ] `load_macro_calendar()` inserts events from JSON file into `macro_events` table
- [ ] `load_macro_calendar()` skips duplicate events (idempotent)
- [ ] `get_upcoming_events()` returns events within N hours, sorted by hours_until
- [ ] `get_upcoming_events()` returns empty list when no events upcoming
- [ ] `get_next_event()` returns closest event or None
- [ ] `fetch_cpi_actual()` returns float from FRED API
- [ ] `fetch_cpi_actual()` returns None when FRED API fails or key missing
- [ ] `classify_surprise()` correctly classifies inflation surprises (5 tiers)
- [ ] `classify_surprise()` correctly classifies NFP surprises (4 tiers)
- [ ] `update_event_actual()` updates macro_events row with actual, surprise, impact
- [ ] All thresholds read from config, not hardcoded
- [ ] All HTTP API calls use `aiohttp` (async)

## Test Strategy

- **Unit tests** with mocked HTTP responses via `AsyncMock` on `_get()` method
- **Mock Fear & Greed API** with known value/classification response
- **Mock FRED API** with known CPI observation response
- **Test macro calendar loading** with a small test JSON file (3-4 events), verify insert count and idempotency
- **Test upcoming events** by pre-inserting events at known times relative to "now" (mock `datetime.now`)
- **Test classify_surprise** with known actual/forecast pairs covering all 5 inflation tiers and 4 NFP tiers
- **Edge cases**: F&G API down, FRED API key missing, empty calendar file, no upcoming events, event exactly at boundary thresholds

## Integration Points

**Provides to:**
- `daily_snapshot.fear_greed` → SS-12 (Signal 4: Mean Reversion — F&G component weight 15%)
- `macro_events` table → SS-13 (Signal 5: Event Risk — macro event proximity component)
- `get_upcoming_events()` / `get_next_event()` → SS-17 (Alert System — macro event alerts)
- `classify_surprise()` → SS-17 (Alert System — post-release surprise notifications)

**Depends on:**
- SS-01: `insert_row()`, `query()` from `custom/utils/db.py`
- SS-02: `CollectorError` from `custom/collectors/spot.py`
- `config/settings.yaml`: `macro_events.*` surprise thresholds
- `config/macro_events_2026.json`: Static macro calendar data
- Environment variable: `FRED_API_KEY` for FRED API access

# SS-17: Alert System

**Master Spec Sections:** §10
**Dependencies:** SS-15
**Estimated LOC:** ~150

## Scope

Evaluates market conditions against alert triggers and manages cooldown to prevent spam. Produces alert dicts with priority, message, and trigger type. Does NOT send alerts (SS-18 handles Telegram delivery) or compute signals (SS-15).

## Deliverables

```
custom/output/
└── alerts.py

tests/
└── test_alerts.py
```

## Key Requirements

1. `check_alerts(db_path, signal, config)` evaluates all alert triggers against current data and returns list of fired alerts.
2. Alert types from §10.1:
   - CRITICAL: gamma flip breach, liquidation cascade >$100M, macro event <2h
   - WARNING: funding extreme, large options expiry >10k BTC <24h, CVD divergence, OI regime change, macro event <24h
   - INFO: signal score crosses ±0.35 or ±0.60, entry condition met, post-macro resume
3. `check_cooldown(alert_type, priority, last_fired, config)` enforces minimum intervals:
   - CRITICAL: 15 min
   - WARNING: 30 min
   - INFO: 2 hours
4. `format_alert(alert)` returns formatted alert message string.
5. Each alert dict: `{"priority": str, "trigger": str, "message": str, "timestamp": str}`.
6. All thresholds from config.

## Acceptance Criteria

- [ ] AC1: CRITICAL alert fires on gamma flip breach
- [ ] AC2: CRITICAL alert fires on liquidation cascade >$100M
- [ ] AC3: CRITICAL alert fires on macro event <2h
- [ ] AC4: WARNING alert fires on extreme funding
- [ ] AC5: INFO alert fires on signal score crossing ±0.35
- [ ] AC6: Cooldown prevents duplicate CRITICAL alerts within 15 min
- [ ] AC7: Cooldown prevents duplicate WARNING alerts within 30 min
- [ ] AC8: Cooldown prevents duplicate INFO alerts within 2 hours
- [ ] AC9: format_alert returns readable string with priority emoji
- [ ] AC10: Returns empty list when no alerts triggered
- [ ] AC11: All thresholds from config

## Integration Points

- **Inputs:** Signal dict from SS-15, DB data (futures_liquidations, gex_data, macro_events, etc.)
- **Outputs:** List of alert dicts consumed by SS-18 (Telegram)

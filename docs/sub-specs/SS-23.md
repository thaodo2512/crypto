# SS-23: Streamlit Dashboard

**Master Spec Sections:** §13
**Dependencies:** SS-15 (Signal Engine), SS-20 (Evaluation & Performance Tracking)
**Estimated LOC:** ~200

## Scope
Streamlit-based read-only dashboard for deep analysis of signal data, options/GEX, price levels, futures metrics, and performance evaluation. Runs on same VPS, accessed via browser. All data sourced from SQLite DB — no write operations.

Does NOT cover: Telegram interface (SS-18), alert delivery (SS-17), or AI analysis (SS-19).

## Deliverables
```
dashboard/
  app.py              # Main Streamlit application with 5 tabs
tests/
  test_dashboard.py   # Unit tests for data-loading helpers
config/
  settings.yaml       # Add dashboard section
```

## Key Requirements

1. Dashboard has 5 tabs: Signals, Options/GEX, Price Levels, Futures, Performance (§13.1)
2. **Signals tab:** Final score gauge, composite signal bars, 30-day signal history line chart
3. **Options/GEX tab:** GEX by strike bar chart, gamma flip vs. price line chart, OI heatmap
4. **Price Levels tab:** Candlestick chart with confluence zones overlaid, level table
5. **Futures tab:** Multi-exchange funding chart, OI chart, basis chart, L/S ratio
6. **Performance tab:** Win rate over time, signal accuracy by regime, component accuracy
7. Charts built with Plotly for interactivity (zoom, hover, export)
8. All data loaded from SQLite via `custom.utils.db.query()`
9. Graceful handling of empty/missing data — show placeholder messages, never crash
10. Lookback days and refresh interval configurable via `config/settings.yaml`

## Test Strategy

- Test data-loading helper functions with seeded SQLite DB
- Test empty-data graceful handling (empty DB returns empty lists/dicts)
- Test config loading for dashboard section
- Mock Streamlit rendering — verify helper outputs, not UI rendering
- Do NOT test Streamlit UI rendering directly (requires browser)

## Integration Points

**Consumes from:**
- `custom.utils.db.query()` — all data reads
- `custom.evaluation.tracker` — `compute_win_rate`, `compute_component_accuracy`, `compute_regime_accuracy`
- SQLite tables: `signals`, `spot_price`, `futures_snapshot`, `options_oi`, `gex_data`, `spot_technicals`

**Provides to:**
- End user via browser (read-only visualization)

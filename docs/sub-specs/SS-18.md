# SS-18: Telegram Bot Interface

**Master Spec Sections:** В§12
**Dependencies:** SS-15, SS-16, SS-17
**Estimated LOC:** ~300

## Scope

Implements Telegram command handlers and message formatting for all user-facing commands. Dispatches to signal engine, trade plan, alert system, regime, and health modules. Formats rich Telegram messages. Does NOT implement AI analysis (SS-19), evaluation/performance (SS-20), or scheduling (SS-21). Stubs are provided for commands that depend on unimplemented modules.

## Key Requirements

1. `TelegramBot` class wraps `python-telegram-bot` and registers all command handlers.
2. Command handlers from В§12.2:
   - `/signal` вЂ” quick signal summary (score, bias, confidence, breakdown)
   - `/breakdown` вЂ” detailed signal component breakdown
   - `/levels` вЂ” confluence zones with strength
   - `/entry long|short` вЂ” trade plan with entry, stop, TPs, size
   - `/risk` вЂ” event risk component breakdown
   - `/regime` вЂ” current regime + weight allocation
   - `/health` вЂ” data source status
   - `/macro` вЂ” upcoming macro events (7 days)
   - `/performance` вЂ” stub (implemented in SS-20)
   - `/ai` вЂ” stub (implemented in SS-19)
3. `format_signal_report(signal)` formats the signal dict into a Telegram message.
4. `format_trade_plan(plan)` formats trade plan for Telegram.
5. `format_levels(zones)` formats confluence zones.
6. `format_alerts(alerts)` formats alert list for sending.
7. `send_daily_report(bot, signal, config)` sends the 08:00 UTC daily report.
8. All formatters return plain text with emoji for Telegram Markdown.

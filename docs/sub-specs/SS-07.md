# SS-07: Calculators (Greeks, GEX, CVD)

**Master Spec Sections:** §3.2 (CVD), §3.4 (Greeks, GEX, Max Pain, IV Skew), §4.3 (GEX Detail)
**Dependencies:** SS-01, SS-04
**Estimated LOC:** ~300

## Scope

Computes derived metrics from raw options and spot data: Black-Scholes Greeks (delta, gamma), Gamma Exposure (GEX) per strike, gamma flip point, max pain strike, IV skew, and CVD z-score normalization. Stores GEX results in the `gex_data` table. This sub-spec does NOT compute signal scores (SS-09, SS-11), confluence zones (SS-08), or trade plans (SS-16) — it only produces the intermediate calculated values that signals consume.

## Key Requirements

### Greeks & GEX (`greeks.py`)

1. `compute_greeks(spot, strike, T, r, iv, option_type)` — Returns dict with `delta` and `gamma` using Black-Scholes via `py_vollib`. Parameters: `spot` (float), `strike` (float), `T` (float, time to expiry in years), `r` (float, risk-free rate), `iv` (float, implied volatility as decimal e.g. 0.65), `option_type` ("call" or "put"). Returns `{"delta": float, "gamma": float}`. On invalid inputs (T <= 0, iv <= 0), returns `{"delta": 0.0, "gamma": 0.0}`.

2. `compute_gex(options_chain, spot)` — Takes list of options_oi rows (from SS-04 `fetch_options_chain()`) and current spot price. For each row with valid call/put IV and OI, computes gamma via Black-Scholes, then: `call_gex = gamma × call_OI × spot² × 0.01`, `put_gex = gamma × put_OI × spot² × 0.01 × (-1)`, `net_gex = call_gex + put_gex`. Returns list of dicts: `[{"strike": float, "call_gex": float, "put_gex": float, "net_gex": float}]`. Uses `r = 0.0` (crypto has no risk-free rate convention).

3. `find_gamma_flip(gex_results)` — Takes output of `compute_gex()`. Sorts by strike ascending, computes cumulative net_gex, finds strike where cumulative sign changes. Returns the gamma flip price (float) or `None` if no flip found.

4. `compute_max_pain(options_chain)` — Takes list of options_oi rows. For each candidate strike K in the chain, computes: `pain(K) = Σ call_OI × max(0, K - strike_i) + Σ put_OI × max(0, strike_i - K)`. Returns the strike K that minimizes pain (float), or `None` if chain is empty.

5. `compute_iv_skew(options_chain, spot)` — Finds ATM options (strike closest to spot) and computes: `skew = ATM_put_IV - ATM_call_IV`. Returns float (positive = more downside fear) or `None` if ATM data unavailable.

6. `store_gex(db_path, gex_results, gamma_flip)` — Writes GEX results to `gex_data` table. Each row: `{date, strike, call_gex, put_gex, net_gex, gamma_flip_price}`.

7. `compute_options_snapshot(db_path, options_chain, spot)` — Orchestrator method that calls `compute_gex()`, `find_gamma_flip()`, `compute_max_pain()`, `compute_iv_skew()`, stores GEX data, and returns a summary dict: `{"gamma_flip": float|None, "max_pain": float|None, "iv_skew": float|None, "total_net_gex": float, "gex_by_strike": list}`.

### CVD Calculator (`cvd.py`)

8. `compute_cvd_zscore(db_path, lookback_days)` — Queries `spot_cvd` table for the last `lookback_days` of `cvd_24h` values. Computes z-score of the latest value against the lookback window: `z = (latest - mean) / std`. Returns float clipped to `[-3.0, +3.0]`, or `0.0` if insufficient data (< 5 data points).

9. `compute_cvd_divergence(db_path)` — Queries latest `spot_cvd` row and latest `spot_price` row. Determines if CVD direction diverges from price direction (e.g. price up but CVD down). Returns dict: `{"cvd_direction": str, "price_direction": str, "is_divergent": bool}`. Uses 24h CVD and 24h price change.

10. `compute_cvd_4h_confirmation(db_path)` — Queries latest `spot_cvd` row, checks if `cvd_4h` sign matches `cvd_24h` sign. Returns `True` if same sign (confirmed), `False` if opposite (contradiction).

## Acceptance Criteria

- [ ] `compute_greeks()` returns delta and gamma for call option using Black-Scholes
- [ ] `compute_greeks()` returns delta and gamma for put option
- [ ] `compute_greeks()` returns zeros when T <= 0 or iv <= 0
- [ ] `compute_gex()` calculates per-strike GEX with correct formula (call positive, put negative)
- [ ] `find_gamma_flip()` finds strike where cumulative net_gex changes sign
- [ ] `find_gamma_flip()` returns None when no sign change exists
- [ ] `compute_max_pain()` returns strike that minimizes total pain
- [ ] `compute_max_pain()` returns None for empty chain
- [ ] `compute_iv_skew()` returns ATM put_IV minus ATM call_IV
- [ ] `store_gex()` writes rows to gex_data table with correct fields
- [ ] `compute_options_snapshot()` returns summary dict with all computed values
- [ ] `compute_cvd_zscore()` returns z-score of latest CVD against lookback window
- [ ] `compute_cvd_zscore()` returns 0.0 when insufficient data
- [ ] `compute_cvd_divergence()` detects when CVD and price move in opposite directions
- [ ] `compute_cvd_4h_confirmation()` returns True when 4h and 24h CVD agree in sign

## Test Strategy

- **Unit tests** for `compute_greeks()` with known Black-Scholes inputs (compare against known outputs)
- **Unit tests** for `compute_gex()` with synthetic options chain data (3-5 strikes)
- **Unit tests** for `find_gamma_flip()` with crafted cumulative GEX that flips at known strike
- **Unit tests** for `compute_max_pain()` with simple 3-strike chain where optimal K is obvious
- **Unit tests** for `compute_iv_skew()` with known ATM put/call IV values
- **Mock `py_vollib`** in GEX tests to isolate logic from Black-Scholes library
- **Use test DB** (from conftest.py `db` fixture) for `store_gex()` and CVD calculator tests
- **Edge cases**: expired options (T <= 0), zero OI, zero IV, single-strike chain, all positive/negative GEX, empty CVD history

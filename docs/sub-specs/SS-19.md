# SS-19: AI Analysis Layer

**Master Spec Sections:** §11
**Dependencies:** SS-15, SS-18
**Estimated LOC:** ~250

## Scope

Builds structured prompts from system data and sends to Claude API for narrative analysis. Supports daily briefing, signal change, macro reaction, and custom question prompts. Handles API errors gracefully. Response language: Vietnamese. Does NOT modify signal scores or trigger trades (AI is advisory only). Rate limit: 10 calls/day max (IMMUTABLE).

## Deliverables

```
custom/ai/
└── analyzer.py

tests/
└── test_ai_analyzer.py
```

## Key Requirements

1. `AIPromptBuilder` class collects all system data and formats into structured prompts.
2. `build_daily_briefing(db_path, config)` builds full daily analysis prompt with all data sections.
3. `build_signal_change(db_path, config, old_score, new_score)` builds prompt for signal threshold crossing.
4. `build_macro_reaction(db_path, config, event, actual, forecast)` builds post-macro prompt.
5. `build_custom_question(db_path, config, question)` builds prompt for `/ai [question]`.
6. `ClaudeAnalyzer` class sends prompts to Anthropic API via aiohttp.
7. `analyze(prompt)` sends to Claude API, returns response text.
8. System prompt defines role, rules, Vietnamese language, Telegram formatting.
9. Rate limiter: max 10 calls/day (IMMUTABLE).
10. On API failure: log error, return fallback message, never crash.

## Acceptance Criteria

- [ ] AC1: Daily briefing prompt includes all data sections (signals, regime, levels, macro, options, futures)
- [ ] AC2: Signal change prompt includes old/new scores and asks for interpretation
- [ ] AC3: Custom question prompt includes user's question with full market context
- [ ] AC4: System prompt enforces Vietnamese language and Telegram formatting
- [ ] AC5: Rate limiter prevents >10 calls/day
- [ ] AC6: API failure returns fallback message (no crash)
- [ ] AC7: AI output never modifies signal scores (advisory only)
- [ ] AC8: All config values from settings.yaml (model, max_tokens, max_daily_calls)

## Integration Points

- **Inputs:** DB data (all tables), signal dict from SS-15, config
- **Outputs:** AI analysis text delivered via SS-18 (Telegram)

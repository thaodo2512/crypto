# SS-24: Dynamic Macro Event Collection

## 1. Purpose

Replace the static `macro_events_2026.json` calendar with a three-layer dynamic
event collection system:

1. **Finnhub Economic Calendar API** — scheduled events with impact classification
2. **FinancialJuice RSS** — breaking news headlines in real-time
3. **Claude Haiku classifier** — AI-based tier classification for RSS headlines

No changes to `event_risk.py` — new events flow into the same `macro_events` table.

## 2. Finnhub Economic Calendar Collector

**File:** `custom/collectors/macro_calendar.py`

### 2.1 Class: `FinnhubCalendarCollector`

- Uses Finnhub `/calendar/economic` endpoint (free tier, no API key required)
- `async fetch_calendar() -> int` — fetches and inserts events
- Lookback: 7 days, lookahead: 14 days (configurable)
- Impact mapping: `high` → Tier 1, `medium` → Tier 2, `low` → Tier 3
- Filters to crypto-relevant events via `finnhub.relevant_events` config list
- Idempotent via `date + event` composite key
- Events tagged with `source='finnhub'`

### 2.2 Scheduling

- Runs every 12 hours (configurable via `finnhub.calendar_interval_hours`)

## 3. News RSS Collector + Haiku Classifier

**Files:** `custom/collectors/news_rss.py`, `custom/ai/classifier.py`

### 3.1 Class: `NewsRSSCollector`

- Parses RSS feed from FinancialJuice
- Deduplicates via `news_rss_seen` table (stores guid)
- Classifies new headlines using `HeadlineClassifier`
- Inserts classified events into `macro_events` with `source='rss'`
- Active window: headline time + `active_hours` from classifier

### 3.2 Class: `HeadlineClassifier`

- Uses `claude-haiku-4-5-20251001` model
- Separate `RateLimiter` (48/day) from Sonnet's 10/day
- Returns `{tier, event_name, active_hours}` or `None` for irrelevant headlines
- Tier classification prompt:
  - Tier 1: >2% BTC move expected
  - Tier 2: 1-2% move
  - Tier 3: <1% move
  - `null`: not crypto-relevant (skipped)

### 3.3 Scheduling

- RSS poll: every 15 minutes (configurable via `news_rss.poll_interval_minutes`)

## 4. Database Changes

### 4.1 `macro_events` table — new column

```sql
source TEXT DEFAULT 'static'
```

Values: `'static'` (JSON file), `'finnhub'` (API), `'rss'` (RSS+Haiku)

### 4.2 New table: `news_rss_seen`

```sql
CREATE TABLE IF NOT EXISTS news_rss_seen (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    guid TEXT UNIQUE NOT NULL,
    title TEXT,
    seen_at TEXT NOT NULL
);
```

### 4.3 Migration

Idempotent `ALTER TABLE macro_events ADD COLUMN source TEXT DEFAULT 'static'`
for pre-SS-24 databases.

## 5. Configuration

```yaml
finnhub:
  calendar_interval_hours: 12
  lookahead_days: 14
  lookback_days: 7
  relevant_events: [...]  # crypto-relevant event keywords

news_rss:
  feed_url: "https://www.financialjuice.com/feed.ashx"
  poll_interval_minutes: 15
  active_window_hours: 4
  max_age_hours: 24

haiku_classifier:
  model: "claude-haiku-4-5-20251001"
  max_tokens: 100
  max_daily_calls: 48
```

## 6. Acceptance Criteria

1. `FinnhubCalendarCollector.fetch_calendar()` returns count of inserted events
2. Finnhub events filtered to crypto-relevant only
3. Duplicate Finnhub events are not re-inserted (idempotent)
4. `NewsRSSCollector.fetch_breaking_news()` returns count of inserted events
5. RSS entries are deduplicated via `news_rss_seen` table
6. `HeadlineClassifier.classify()` returns tier/event_name/active_hours or None
7. Irrelevant headlines (tier=null) are not inserted into macro_events
8. All new events have correct `source` column value
9. Graceful degradation: failures return 0, log error, don't crash
10. Haiku rate limiter is separate from Sonnet's (48/day vs 10/day)
11. Health monitor tracks both new data sources
12. Static JSON events still work unchanged (backward compatible)

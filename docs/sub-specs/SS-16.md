# SS-16: Trade Plan Generator

**Master Spec Sections:** §7
**Dependencies:** SS-08, SS-15
**Estimated LOC:** ~300

## Scope

Generates a complete trade plan from the signal engine output and confluence zones. Includes entry gate validation, position sizing (fixed fractional risk), stop loss placement, scaled take-profit levels (TP1/TP2/TP3), and time stop. Does NOT execute trades, send alerts (SS-17), or interface with Telegram (SS-18).

## Key Requirements

1. `check_entry_gate(signal, zones, config)` validates 3 gates:
   - Gate 1: strength != WEAK and confidence != LOW
   - Gate 2: nearest confluence zone exists
   - Gate 3: (informational) confirmations available
   Returns pass/reject/wait with reason.
2. `compute_position_size(portfolio_usd, signal, config)` calculates position size:
   - HIGH confidence + STRONG → 2% risk
   - HIGH or STRONG (one of two) → 1.5% risk
   - MEDIUM confidence → 1% risk
   - LOW confidence → 0.5% risk
   - Leverage capped at 3x (IMMUTABLE)
   - position_size = risk_amount / stop_distance
3. `compute_stop_loss(direction, zones, spot, config)` places stop below/above nearest support/resistance zone + buffer (0.3–0.5%).
4. `compute_take_profits(direction, entry, stop, zones, config)` computes scaled exits:
   - TP1: 50% at nearest resistance (min 1.5R)
   - TP2: 30% at next resistance (target 2–3R)
   - TP3: 20% trailing stop (-2% from high)
5. `generate_trade_plan(signal, zones, spot, portfolio_usd, config)` orchestrator.
6. `store_trade_plan(db_path, plan)` writes to `trades` table.
7. All risk limits from config. max_risk=2%, max_leverage=3x, min_rr=1.5 are IMMUTABLE.

## Acceptance Criteria

- [ ] AC1: Gate 1 rejects WEAK strength or LOW confidence
- [ ] AC2: Gate 2 rejects when no confluence zone exists
- [ ] AC3: Position size never exceeds 2% portfolio risk (IMMUTABLE)
- [ ] AC4: Leverage never exceeds 3x (IMMUTABLE)
- [ ] AC5: Stop loss placed below support zone + buffer
- [ ] AC6: TP1 at nearest resistance, minimum 1.5R
- [ ] AC7: TP2 at next resistance zone
- [ ] AC8: Scaled exit: 50%/30%/20% split
- [ ] AC9: Plan stored in trades table
- [ ] AC10: Returns None when entry gate fails
- [ ] AC11: All thresholds from config

## Integration Points

- **Inputs:** Signal dict from SS-15, confluence zones from SS-08, spot price
- **Outputs:** Trade plan dict stored in `trades` table, consumed by SS-18 (Telegram)

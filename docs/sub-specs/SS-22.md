# SS-22: Backtesting Framework

**Master Spec Sections:** §15
**Dependencies:** SS-15
**Estimated LOC:** ~200

## Scope

Custom walk-forward validator for composite signal backtesting. Trains on 60-day windows, tests on 30-day windows, sliding forward 30 days per iteration. Only optimizes regime weights; signal thresholds remain fixed. Tracks win rate, average R:R, Sharpe ratio, max drawdown, accuracy by regime. Does NOT replace Freqtrade's built-in backtesting or modify live signal parameters.

## Key Requirements

1. `generate_walk_forward_periods(start_date, end_date, train_days, test_days, slide_days)` generates list of (train_start, train_end, test_start, test_end) tuples.
2. `replay_signals(db_path, config, start_date, end_date)` replays signal computation on historical data for a date range, returning list of signal results.
3. `evaluate_period(signals, db_path, horizon_hours)` computes metrics for a test period:
   - Win rate: % of signals where price moved in predicted direction
   - Average R:R
   - Max drawdown
   - Signal count
4. `compute_backtest_metrics(results)` aggregates across all walk-forward periods:
   - Overall win rate, Sharpe ratio, max drawdown
   - Accuracy by regime
   - Component contribution
5. `run_backtest(db_path, config, start_date, end_date)` orchestrator.
6. Walk-forward: train=60 days, test=30 days, slide=30 days (from config).
7. Only regime weights are optimized. Signal thresholds remain fixed.

## Acceptance Criteria

- [ ] AC1: Walk-forward periods generated correctly (60/30/30 default)
- [ ] AC2: Periods do not overlap in test windows
- [ ] AC3: Signal replay produces results for date range
- [ ] AC4: Win rate computed correctly (direction match within horizon)
- [ ] AC5: Max drawdown tracks largest peak-to-trough
- [ ] AC6: Accuracy segmented by regime
- [ ] AC7: Returns empty metrics on insufficient data (no crash)
- [ ] AC8: All window sizes from config

## Integration Points

- **Inputs:** Historical data in DB, config settings, signal engine (SS-15)
- **Outputs:** Backtest metrics dict, consumed by evaluation reports and dashboard

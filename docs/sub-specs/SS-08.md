# SS-08: Confluence Zone Calculator

**Master Spec Sections:** §7.2
**Dependencies:** SS-01, SS-07
**Estimated LOC:** ~150

## Scope

Detects price areas where 3 or more independent support/resistance levels cluster within ±0.5%. Sources include options walls, gamma flip, max pain, EMAs, VWAP, liquidation clusters, and round numbers. Classifies zones as support (below spot) or resistance (above spot) and ranks by strength. Does NOT generate trade plans (SS-16), entry triggers, or position sizing.

## Key Requirements

1. `collect_levels(db_path, spot, snapshot)` gathers all level sources into a unified list of `{"price": float, "type": str, "source": str}` dicts.
2. Level sources (from §7.2):
   - Call/Put Walls: top OI strikes from `options_oi`
   - Gamma Flip Price: from options snapshot or `gex_data`
   - Max Pain: from options snapshot
   - EMA 21/55/200: from `spot_technicals`
   - VWAP: from `spot_technicals`
   - Liquidation Clusters: from `futures_liquidations` (recent high liquidation prices)
   - Round Numbers: nearest $5k round numbers to spot (e.g., $95k, $100k, $105k)
3. `find_confluence_zones(levels, spot, config)` implements the clustering algorithm:
   - Sort levels by price ascending
   - For each level, find all other levels within ±`confluence_zone_width_pct`% of its price
   - If count ≥ `confluence_min_levels` (default 3) → create a zone
   - Zone center = mean of clustered prices, components = list of source types
   - Merge overlapping zones
   - Classify: support (center < spot) or resistance (center > spot)
4. `compute_confluence_zones(db_path, config, spot, snapshot)` is the orchestrator.
5. All thresholds from `config/settings.yaml` under `trade_plan.confluence_*`.
6. Returns list of zone dicts sorted by distance from spot.

## Acceptance Criteria

- [ ] AC1: Collects levels from all 8 sources (call walls, put walls, gamma flip, max pain, EMA 21/55/200, VWAP, round numbers)
- [ ] AC2: Zones require ≥3 levels within ±0.5% (from config)
- [ ] AC3: Zone center = mean of clustered level prices
- [ ] AC4: Zones classified as "support" (below spot) or "resistance" (above spot)
- [ ] AC5: Zones ranked by strength (number of overlapping levels)
- [ ] AC6: Overlapping zones are merged
- [ ] AC7: Round numbers ($5k increments) included as level source
- [ ] AC8: Returns empty list when insufficient data (no crash)
- [ ] AC9: All thresholds from config

## Test Strategy

- Mock DB data with known levels at specific prices to create predictable clusters
- Test each level source collection independently
- Test zone detection with exactly 3 levels at same price
- Test support vs resistance classification
- Test merge of overlapping zones
- Test empty/insufficient data returns empty list
- Test round number generation

## Integration Points

- **Inputs:** `spot_technicals`, `options_oi`, `gex_data`, `futures_liquidations` tables; options snapshot dict from SS-07
- **Outputs:** List of confluence zone dicts consumed by SS-16 (Trade Plan Generator)

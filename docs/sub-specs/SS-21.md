# SS-21: Scheduler & Main Entry Point

**Master Spec Sections:** §9
**Dependencies:** SS-02→SS-06, SS-15, SS-17, SS-18
**Estimated LOC:** ~150

## Scope

APScheduler-based main loop that orchestrates all data collection, signal computation, alerting, and reporting at the intervals defined in §9. Supports standard and elevated-risk schedules. Entry point for `python main.py`. Does NOT implement individual collectors or signals (those are in SS-02→SS-15).

## Deliverables

```
main.py (modify existing)

custom/
└── scheduler.py

tests/
└── test_scheduler.py
```

## Key Requirements

1. `SignalBotScheduler` class wraps APScheduler with all job definitions.
2. Standard intervals from §9.1:
   - Every 2 min: price fetch + alert checks
   - Every 15 min: orderbook, futures OI, basis, taker ratio
   - Every 1 hour: funding, L/S ratio, CVD, whale trades, liquidations
   - Every 4 hours: options scan, signal recompute
   - Daily 08:00 UTC: full report, technicals, regime, F&G, zones, macro, maintenance
3. `check_elevated_risk(event_risk)` determines if elevated schedule needed.
4. Elevated intervals (event_risk > 0.7): price 30s, futures 5min, options 30min, signals 30min.
5. `start()` and `stop()` methods for lifecycle management.
6. All intervals from `config/settings.yaml` under `scheduling`.

## Acceptance Criteria

- [ ] AC1: Standard schedule registers jobs at correct intervals
- [ ] AC2: Elevated risk tightens intervals when event_risk > 0.7
- [ ] AC3: Daily job scheduled at 08:00 UTC
- [ ] AC4: start() begins scheduler, stop() shuts it down
- [ ] AC5: All intervals from config
- [ ] AC6: Elevated risk reverts to standard when risk drops
- [ ] AC7: Scheduler handles job errors without crashing

## Integration Points

- **Inputs:** Config (scheduling intervals), event risk from SS-13
- **Outputs:** Triggers all data collection (SS-02→SS-06), signal computation (SS-15), alerts (SS-17), Telegram (SS-18)

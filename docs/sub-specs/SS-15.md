# SS-15: Signal Engine (Final Score)

**Master Spec Sections:** §6
**Dependencies:** SS-09, SS-10, SS-11, SS-12, SS-13, SS-14
**Estimated LOC:** ~200

## Scope

Combines the 4 directional signal scores with regime-based adaptive weights into a single final score. Applies consensus check (agreement multiplier), event risk confidence penalty, classifies signal strength/bias/confidence. Stores result in `signals` table. Does NOT compute individual signals (SS-09→SS-13), regime detection (SS-14), trade plans (SS-16), or alerts (SS-17).

## Key Requirements

1. `compute_raw_score(signals, weights)` computes weighted sum of 4 directional signals: `raw = Σ(signal_i × weight_i)`. Range [-1, +1].
2. `check_consensus(signals, config)` counts bullish (>0.15), bearish (<-0.15), neutral signals. Returns consensus type and multiplier:
   - 3+ same direction → STRONG_CONSENSUS (×1.3)
   - 2 same, 0 opposite → MODERATE_CONSENSUS (×1.1)
   - 2+ positive AND 2+ negative → CONFLICT (×0.5)
   - Otherwise → MIXED (×0.8)
3. `apply_event_risk_penalty(score, event_risk, config)` applies confidence penalty:
   - event_risk > 0.7 → ×0.4
   - event_risk 0.4–0.7 → ×0.7
   - event_risk < 0.4 → ×1.0
4. `classify_signal(score, config)` returns bias (LONG/SHORT), strength (NEUTRAL/WEAK/MODERATE/STRONG).
5. `compute_confidence(consensus_multiplier, event_risk, regime, final_score, config)` computes confidence level (HIGH/MEDIUM/LOW) from 4 boolean factors.
6. `compute_final_signal(db_path, config, prev_weights)` is the orchestrator — calls all 5 signals, regime, then assembles the final output.
7. `store_signal(db_path, result)` writes to the `signals` table.
8. All thresholds from `config/settings.yaml`.

## Acceptance Criteria

- [ ] AC1: Raw score = weighted sum of 4 directional signals
- [ ] AC2: STRONG_CONSENSUS when 3+ signals same direction (×1.3)
- [ ] AC3: CONFLICT when 2+ bullish AND 2+ bearish (×0.5)
- [ ] AC4: Event risk >0.7 applies ×0.4 penalty
- [ ] AC5: Event risk 0.4–0.7 applies ×0.7 penalty
- [ ] AC6: Signal classified as NEUTRAL when |score| < 0.15
- [ ] AC7: Signal classified as STRONG when |score| > 0.60
- [ ] AC8: Confidence = HIGH when ≥3/4 factors true
- [ ] AC9: Confidence = LOW when <2/4 factors true
- [ ] AC10: Final score clipped to [-1, +1]
- [ ] AC11: Output matches §6.2 schema (timestamp, bias, strength, confidence, regime, etc.)
- [ ] AC12: All thresholds from config
- [ ] AC13: Returns neutral on failure (score=0, NEUTRAL, LOW confidence)

## Test Strategy

- Test raw score computation with known weights and signals
- Test all 4 consensus types with specific signal combinations
- Test event risk penalty at each tier boundary
- Test signal classification at each threshold
- Test confidence computation with various factor combinations
- Test full pipeline with mocked signal/regime outputs
- Test failure mode returns neutral

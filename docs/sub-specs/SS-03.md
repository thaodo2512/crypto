## Scope

Collects futures market data from Binance Futures API (primary) with cross-validation from Bybit and OKX: funding rates, open interest, L/S ratios, taker buy/sell ratio, basis/premium, and liquidation data (via Coinglass). Computes derived metrics: OI-weighted funding rate, futures basis, OI-price regime, and OI change percentages. Stores results in `futures_snapshot`, `futures_liquidations`, and `futures_oi_price` tables. This sub-spec does NOT compute Signal 2 (Leverage Positioning) or any composite scores â€” it only fetches and stores data.

## Key Requirements

1. `FuturesCollector` class with async methods, using `aiohttp` for all HTTP calls. Reuses `CollectorError` from `custom/collectors/spot.py`.
2. `fetch_snapshot()` collects a full futures snapshot from multiple endpoints and stores one row in `futures_snapshot`:
   - Funding rates from Binance (`/fapi/v1/fundingRate`), Bybit (`/v5/market/funding/history`), OKX (`/api/v5/public/funding-rate`).
   - Open interest from Binance (`/fapi/v1/openInterest`), Bybit (`/v5/market/open-interest`), OKX (`/api/v5/public/open-interest`).
   - L/S ratios from Binance (`/futures/data/topLongShortPositionRatio`, `/futures/data/globalLongShortAccountRatio`).
   - Taker buy/sell ratio from Binance (`/futures/data/takerlongshortRatio`).
   - Futures price + basis from Binance (`/fapi/v1/premiumIndex`).
3. **Aggregated Funding Rate:** `funding_weighted_avg = ÎŁ(funding_i Ă— oi_i) / ÎŁ(oi_i)` where weight = each exchange's OI proportion.
4. **Futures Basis:** `basis_pct = (futures_price - spot_price) / spot_price Ă— 100`. Annualized premium: `basis_pct Ă— (365 / days_to_expiry)` (use 1 for perp contracts since funding settles every 8h, approximate annualized = `funding_rate Ă— 3 Ă— 365`).
5. **OI Change Percentages:** `oi_change_Xh_pct` computed by comparing current `oi_total_usd` against the value from X hours ago (queried from `futures_snapshot` table). Returns `None` if no historical data available.
6. `fetch_oi_price_regime()` classifies the OI-price relationship and stores in `futures_oi_price`:
   - `NEW_LONGS`: OIâ†‘ AND Priceâ†‘
   - `NEW_SHORTS`: OIâ†‘ AND Priceâ†“
   - `LONG_CLOSING`: OIâ†“ AND Priceâ†“
   - `SHORT_CLOSING`: OIâ†“ AND Priceâ†‘
7. `fetch_liquidations()` fetches liquidation data from Coinglass free tier and stores in `futures_liquidations`. Computes `liq_ratio = long_liq_usd / total_liq_usd` (0.5 = balanced, >0.5 = more longs liquidated). Returns `None` values if Coinglass is unavailable (free tier fallback).
8. On HTTP error from any exchange, log the error and set that exchange's values to `None` in the snapshot â€” do NOT fail the entire snapshot. Only raise `CollectorError` if ALL primary sources fail.
9. Bybit base URL: `https://api.bybit.com`. OKX base URL: `https://www.okx.com`. Binance Futures base URL: `https://fapi.binance.com`. Coinglass base URL: `https://open-api.coinglass.com`.
10. Coinglass rate limit: ~100 req/day. Only call 2-3 times per day.

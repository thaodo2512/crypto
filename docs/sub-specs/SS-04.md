# SS-04: Options Data Collector (Deribit)

**Master Spec Sections:** §3.4, §9.1, §9.3, §14.3
**Dependencies:** SS-01
**Estimated LOC:** ~200

## Scope

Collects options market data from the Deribit public API: enumerates all active BTC option instruments, fetches ticker data (OI, IV, volume) per strike/expiry, fetches the BTC index price, collects the DVol index, and detects large/unusual trades. Stores raw options chain data in `options_oi`, large trades in `options_large_trades`, and DVol + put/call ratios in `daily_snapshot`. This sub-spec does NOT compute Greeks, GEX, gamma flip, max pain, or IV skew — those belong to SS-07 (Calculators).

## Key Requirements

1. `OptionsCollector` class with async methods, using `aiohttp` for all HTTP calls. Reuses `CollectorError` from `custom/collectors/spot.py`.
2. `fetch_instruments()` enumerates all active BTC option contracts via `GET /public/get_instruments?currency=BTC&kind=option`. Returns list of instrument dicts with parsed `strike`, `expiry`, `option_type` (call/put), `instrument_name`. Filters to only active (`is_active=True`) instruments.
3. `fetch_options_chain()` fetches ticker data for all active instruments and stores one row per strike/expiry in `options_oi`:
   - For each instrument: OI (`open_interest`), IV (`mark_iv`), volume (`stats.volume`), mark price.
   - Aggregates calls and puts at same strike/expiry into single rows with `call_oi`, `put_oi`, `call_iv`, `put_iv`, `call_volume`, `put_volume`.
   - Uses `GET /public/get_book_summary_by_currency?currency=BTC&kind=option` for efficient batch fetch (one call vs hundreds of individual ticker calls).
4. `fetch_index_price()` fetches BTC index price via `GET /public/get_index_price?index_name=btc_usd`. Returns `float`. This is the spot reference used for all options calculations.
5. `fetch_dvol()` fetches the Deribit DVol (implied volatility index) via `GET /public/ticker?instrument_name=BTC-DVOL`. Returns `float` (annualized IV percentage).
6. `fetch_large_trades()` fetches recent trades via `GET /public/get_last_trades_by_currency?currency=BTC&kind=option&count=100`. Filters trades where `amount × mark_price × index_price > whale_threshold_usd` (from config: `spot_data.whale_trade_threshold_usd`). Stores qualifying trades in `options_large_trades` table.
7. `fetch_snapshot()` orchestrates a full options scan: calls `fetch_instruments()`, `fetch_options_chain()`, `fetch_index_price()`, `fetch_dvol()`, `fetch_large_trades()` — stores all data and returns a summary dict with `total_instruments`, `total_call_oi`, `total_put_oi`, `put_call_ratio_oi`, `put_call_ratio_volume`, `dvol`, `index_price`, `large_trades_count`.
8. Deribit base URL: `https://www.deribit.com/api/v2`. Rate limit: 20 req/s — well within budget during 4-hour scan (~600 calls/day estimated).
9. On HTTP error, log and return `None` for individual endpoints. Only raise `CollectorError` if the full snapshot fails (instruments endpoint down = nothing to scan).
10. All thresholds read from `config/settings.yaml`, not hardcoded.

## Acceptance Criteria

- [ ] `fetch_instruments()` returns list of active BTC option instruments with strike, expiry, option_type parsed
- [ ] `fetch_instruments()` filters out inactive instruments
- [ ] `fetch_options_chain()` stores one row per strike/expiry in `options_oi` with call/put OI, IV, volume
- [ ] `fetch_options_chain()` aggregates call and put data at same strike/expiry into single row
- [ ] `fetch_index_price()` returns BTC index price as float
- [ ] `fetch_dvol()` returns DVol index as float
- [ ] `fetch_large_trades()` filters trades above whale threshold and stores in `options_large_trades`
- [ ] `fetch_large_trades()` stores instrument, direction, amount, price, iv, value_usd
- [ ] `fetch_snapshot()` orchestrates full scan and returns summary dict
- [ ] `fetch_snapshot()` computes put/call ratio by OI and by volume
- [ ] Deribit API errors for individual endpoints return None (graceful degradation)
- [ ] Total failure of instruments endpoint raises `CollectorError`
- [ ] All thresholds read from config, not hardcoded
- [ ] All API calls use `aiohttp` (async)

## Test Strategy

- **Unit tests** with mocked HTTP responses via `AsyncMock` on `_get()` method
- **Mock `get_instruments`** response with 4-6 sample instruments (mix of calls/puts at 2-3 strikes, 1-2 expiries)
- **Mock `get_book_summary_by_currency`** with matching ticker data for the sample instruments
- **Test aggregation**: verify that calls and puts at same strike/expiry are merged into one `options_oi` row
- **Test large trade filtering**: provide trades above and below threshold, verify only large ones stored
- **Test put/call ratio**: known OI/volume values → verify ratios computed correctly
- **Edge cases**: empty instruments list, Deribit down (instruments endpoint fails), DVol unavailable, zero OI at a strike, no large trades found

# SS-01: Foundation & Infrastructure

**Master Spec Sections:** §2, §14, §17, §18
**Dependencies:** None
**Estimated LOC:** ~250

## Scope

Sets up the project skeleton: database initialization (all tables from §14), configuration loading (settings.yaml), utility modules (DB helpers, normalizers, health monitor base), and the Freqtrade strategy base class. This sub-spec does NOT implement any data collectors, signal logic, or Telegram commands — it provides the foundation that all other modules build on.

## Key Requirements

1. Database module (`db.py`) must create all tables from §14 (spot_price, spot_cvd, spot_orderbook, spot_whale_trades, spot_technicals, futures_snapshot, futures_liquidations, futures_oi_price, options_oi, gex_data, options_large_trades, daily_snapshot, signals, trades, macro_events, ai_analysis_log, level_outcomes) — 17 tables total.
2. Database module provides helper functions: `init_db()`, `get_db()` (connection factory), `insert_row(table, data)`, `query(sql, params)`, `get_latest(table, n)`.
3. Config loader reads `config/settings.yaml` and provides typed access to all parameters. Missing config keys raise `KeyError` with clear message.
4. Normalizer module provides: `z_score(value, mean, std) → float`, `clip_score(score, min=-1.0, max=1.0) → float`, `normalize_range(value, min_val, max_val) → float` in [-1, +1].
5. `z_score()` returns 0.0 when `std` is 0 (avoid division by zero).
6. `clip_score()` uses `numpy.clip()` internally.
7. All `__init__.py` files are empty (no implicit imports).
8. `main.py` skeleton imports config and calls `init_db()` on startup.
9. `conftest.py` provides `db` fixture (in-memory SQLite) and `config` fixture (dict with test values).
10. `composite_strategy.py` extends Freqtrade's `IStrategy` with placeholder `populate_indicators()`, `populate_entry_trend()`, `populate_exit_trend()`.

## Acceptance Criteria

- [ ] `init_db()` creates all 17 tables without errors
- [ ] `init_db()` is idempotent (calling twice doesn't error or duplicate tables)
- [ ] `insert_row()` inserts a row and returns the row ID
- [ ] `query()` returns list of dicts for SELECT statements
- [ ] `get_latest()` returns N most recent rows ordered by timestamp/date
- [ ] `z_score(10, 5, 2)` returns `2.5`
- [ ] `z_score(10, 5, 0)` returns `0.0` (safe division)
- [ ] `clip_score(1.5)` returns `1.0`
- [ ] `clip_score(-2.0)` returns `-1.0`
- [ ] `clip_score(0.5)` returns `0.5` (no clipping needed)
- [ ] `normalize_range(75, 50, 100)` returns `0.0` (midpoint → 0)
- [ ] Config loads from YAML file and provides dict-like access
- [ ] Missing config key raises `KeyError`
- [ ] `main.py` runs without errors (imports + init_db)
- [ ] All `__init__.py` files exist and are empty

## Test Strategy

- **Unit tests** for `db.py`: use in-memory SQLite (`:memory:`), test CRUD operations
- **Unit tests** for `normalizers.py`: pure math functions, test with known inputs/outputs
- **Golden tests**: hand-calculated z-scores and normalizations
- **Edge cases**: zero std dev, extreme values, empty database, duplicate init_db calls
- **No mocks needed**: these are pure utilities with no external dependencies

## Integration Points

**Provides to all other modules:**
- `get_db()` — database connection
- `init_db()` — schema creation
- `insert_row()`, `query()`, `get_latest()` — data access
- `z_score()`, `clip_score()`, `normalize_range()` — signal normalization
- `conftest.py` fixtures — test infrastructure

**Depends on:**
- Nothing (this is the foundation)

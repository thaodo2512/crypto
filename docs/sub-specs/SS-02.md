# SS-02: Spot Data Collector

**Master Spec Sections:** §3.2, §9.1, §9.3
**Dependencies:** SS-01
**Estimated LOC:** ~200

## Scope

Collects all spot market data from Binance Spot API: OHLCV candles, order book snapshots, aggregated trades (for CVD and whale detection), and technical indicators. Stores results in the 5 spot tables (`spot_price`, `spot_cvd`, `spot_orderbook`, `spot_whale_trades`, `spot_technicals`). This sub-spec does NOT compute composite signals, regime detection, or scheduling — it only fetches and stores raw + derived spot data.

## Key Requirements

1. `SpotCollector` class with async methods for each data type, using `aiohttp` for all HTTP calls.
2. `fetch_price()` calls `GET /api/v3/klines?symbol=BTCUSDT&interval=1h&limit=1` and stores OHLCV data in `spot_price` table.
3. `fetch_orderbook()` calls `GET /api/v3/depth?symbol=BTCUSDT&limit={depth_levels}` where `depth_levels` comes from `config.spot_data.orderbook_depth_levels` (default 20). Computes order book imbalance: `(total_bid_depth - total_ask_depth) / (total_bid_depth + total_ask_depth)`. Range: [-1, +1]. Computes spread in basis points: `(best_ask - best_bid) / best_bid * 10000`. Stores in `spot_orderbook` table.
4. `fetch_trades()` calls `GET /api/v3/aggTrades?symbol=BTCUSDT&limit=1000`. From the raw trades, computes:
   - **CVD**: buy_volume (where `isBuyerMaker == false`) minus sell_volume (where `isBuyerMaker == true`) over 1h, 4h, and 24h windows. Stores in `spot_cvd` table.
   - **Whale trades**: filters trades where `price * quantity >= whale_trade_threshold_usd` from `config.spot_data.whale_trade_threshold_usd` (default $100,000). Stores each in `spot_whale_trades` table.
5. `fetch_technicals()` calls `GET /api/v3/klines?symbol=BTCUSDT&interval=1d&limit=210` (enough for EMA-200 + warmup). Computes via `pandas-ta`: RSI(14), EMA(21), EMA(55), EMA(200), VWAP, Bollinger Bands(20, 2), ADX(14). Computes `volume_sma_20` and `volume_ratio = current_volume / volume_sma_20`. Stores in `spot_technicals` table.
6. All API calls use Binance base URL `https://api.binance.com`.
7. All methods accept a `db_path: str` parameter for database storage.
8. On HTTP error (non-200 status), log the error and raise `CollectorError` (custom exception defined in this module).
9. On network timeout, log the error and raise `CollectorError`.
10. Rate limit awareness: Binance Spot allows 1,200 req/min — collector must not exceed this. Add configurable timeout (default 10s per request).
11. Binance aggTrades uses `isBuyerMaker` field: `false` = buy aggression, `true` = sell aggression.

## Acceptance Criteria

- [ ] `fetch_price()` returns OHLCV dict and stores row in `spot_price` table
- [ ] `fetch_orderbook()` computes imbalance in range [-1, +1] and stores in `spot_orderbook`
- [ ] `fetch_orderbook()` computes spread in basis points correctly
- [ ] Order book imbalance returns 0.0 when bid_depth == ask_depth
- [ ] `fetch_trades()` computes CVD (buy_volume - sell_volume) and stores in `spot_cvd`
- [ ] `fetch_trades()` detects whale trades above threshold and stores in `spot_whale_trades`
- [ ] No whale trades stored when all trades are below threshold
- [ ] `fetch_technicals()` computes RSI, EMA(21/55/200), VWAP, BB, ADX and stores in `spot_technicals`
- [ ] `fetch_technicals()` computes `volume_ratio = current_volume / volume_sma_20`
- [ ] HTTP error (e.g., 500) raises `CollectorError` with status code in message
- [ ] Network timeout raises `CollectorError`
- [ ] All thresholds read from config, not hardcoded
- [ ] All API calls use `aiohttp` (async)

## Test Strategy

- **Unit tests** with mocked HTTP responses (`aiohttp` mocked via `aioresponses` or manual mock)
- **Mock all Binance API calls** — provide fixture JSON responses matching real Binance response format
- **Test CVD calculation** with known trade data: e.g., 3 buys + 2 sells → verify cvd = buy_vol - sell_vol
- **Test whale detection** with trades above and below threshold
- **Test order book imbalance** with symmetric (→ 0.0) and asymmetric books
- **Test technical indicators** with known candle data (at least 210 candles for EMA-200)
- **Edge cases**: empty trade list, empty order book, API returning partial data
- **Error cases**: HTTP 429 (rate limit), HTTP 500, connection timeout

# SS-14: Regime Detection & Adaptive Weights

**Master Spec Sections:** §5
**Dependencies:** SS-01, SS-02
**Estimated LOC:** ~150

## Scope

Detects current market regime using ADX(14) and Bollinger Band Width percentile (90-day lookback). Regime determines adaptive signal weights. Includes EMA smoothing to prevent weight whipsawing. Does NOT compute individual signal scores (SS-09→SS-13) or the final combined score (SS-15).

```
custom/regime/

tests/

```

## Key Requirements

1. `detect_regime(adx, bb_width_percentile, config)` classifies into: STRONG_TREND, MODERATE_TREND, WIDE_RANGE, TIGHT_RANGE, TRANSITIONAL using config thresholds.
2. `get_regime_weights(regime, config)` returns weight dict for the 4 directional signals based on regime from config `adaptive_weights.<regime>`.
3. `smooth_weights(new_weights, prev_weights, smoothing_factor)` applies EMA smoothing: `w_t = smoothing × new + (1 - smoothing) × prev`.
4. `compute_bb_width_percentile(db_path, lookback_days)` queries `spot_technicals` for BB width history, computes percentile of latest value.
5. Regime classification rules from §5.1 using ADX and BB Width %ile thresholds from config.

## Acceptance Criteria

- [ ] STRONG_TREND detected when ADX > 30 and BB %ile > 60
- [ ] MODERATE_TREND detected when ADX > 25 and BB %ile > 40
- [ ] WIDE_RANGE detected when ADX < 25 and BB %ile < 50
- [ ] TIGHT_RANGE detected when ADX < 20 and BB %ile < 30
- [ ] TRANSITIONAL for ambiguous combinations
- [ ] Regime weights match config values
- [ ] Weight smoothing applies EMA correctly
- [ ] BB width percentile computes from history
- [ ] All thresholds from config
